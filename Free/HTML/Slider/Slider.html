Slider / carrossel

Um slider (ou carrossel) é um componente de interface que permite exibir vários conteúdos no mesmo espaço visual, permitindo que o usuário navegue entre eles horizontalmente ou verticalmente.

Ele é muito usado para:

  • Galerias de imagens
  • Cards de produtos
  • Destaques de conteúdo
  • Portfólios

Durante muitos anos, sliders foram implementados com JavaScript, recriando manualmente comportamentos que o navegador já possui, como rolagem, toque, foco e inércia.

Hoje, os navegadores modernos oferecem recursos nativos que permitem criar sliders performáticos, acessíveis e responsivos usando apenas HTML e CSS.

O slider moderno

Na web moderna, um slider não é uma animação que move elementos com transform. Ele é, na verdade, um container rolável que utiliza a rolagem nativa do navegador.

O que define o comportamento de "encaixar" os itens é a propriedade scroll-snap, que instrui o navegador a alinhar automaticamente os elementos em pontos específicos durante a rolagem.

Isso significa que o slider moderno:

  • Funciona com mouse, toque, trackpad e teclado
  • Tem inércia e suavidade nativas
  • É acessível por padrão
  • Não precisa recriar comportamento com JavaScript

Neste modelo, o slider não move os elementos manualmente. O que acontece é que o container possui rolagem horizontal (overflow-x: auto), e cada slide define um ponto de encaixe usando scroll-snap-align.

A propriedade scroll-snap-type no contêiner instrui o navegador a alinhar automaticamente o scroll nesses pontos, criando o efeito de "slides".

Esse método é mais rápido, mais acessível e mais compatível com dispositivos de toque do que sliders implementados com transformações em JavaScript.

Personalizando os nossos slides

Até aqui criamos um slider funcional usando apenas rolagem e scroll-snap. Agora vamos evoluir esse componente adicionando controles visuais, como indicadores (bolinhas) e navegação, sem usar JavaScript.

A ideia é mostrar que o próprio sistema de navegação do navegador pode ser usado como estado da interface.

Dot Slider

Slider com "bolinhas" para navegação

Como funciona: usamos scroll-snap-type no container e scroll-snap-align em cada slide. As bolinhas são geradas pelo navegador (via ::scroll-marker-group e ::scroll-marker), permitindo navegação sem JavaScript.

Limitações:

  • Funciona apenas em navegadores que suportam os pseudo-elementos experimentais ::scroll-marker e ::scroll-marker-group (Chrome/Edge).
  • Firefox, Safari e outros navegadores ainda não suportam visualmente as bolinhas.
  • Não é possível personalizar a interação das bolinhas de forma avançada sem JS.
  • Não há suporte nativo para indicar qual slide está ativo em todos os navegadores.

Slider com setas

Este slider utiliza **scroll nativo** do navegador, mas personalizamos os **botões de navegação (setas)** usando a pseudo-classe `::scroll-button`. A barra de rolagem é escondida para que o usuário navegue apenas pelas setas.

Funciona de forma similar ao slider padrão:

  • Rolagem horizontal nativa
  • Snap automático em cada slide
  • Botões fixos para navegar para a esquerda ou direita

Limitações:

  • Compatibilidade limitada: principalmente Firefox e Chrome funcionam bem, Safari desktop e mobile ainda possuem restrições.
  • Não é tão customizável quanto sliders feitos com JS.
  • Em dispositivos móveis, a aderência aos botões de scroll nativo ainda é baixa.
